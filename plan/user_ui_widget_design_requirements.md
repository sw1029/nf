# User UI 위젯/레이아웃 후속 요구사항

## 목적
- 이번 배포에서는 비디자인 범위(계약 정합성/상태 표출/엔진 보완)만 반영했다.
- 아래 항목은 **신규 위젯/레이아웃 변경**이 필요하므로 후속 과제로 분리한다.

## 후속 요구사항
1. 작업 패널(Queue/Running/Done) 위젯
   - job 목록, 현재 단계, 최근 이벤트 로그를 한 화면에서 표시.
   - 항목별 `취소`, `재시도` 액션 제공.

2. 큐/혼잡도 가시화 위젯
   - heavy job 동시 실행 제한 상태(`running/max`)를 명시.
   - queued 건수와 예상 대기 상태를 사용자가 확인 가능해야 함.

3. 메모리 압력 일시정지 배지
   - `PAUSED_DUE_TO_MEMORY_PRESSURE` 수신 시 시각 배지/툴팁 제공.
   - RSS/제한 메모리와 재개 조건을 설명.

4. 결과 카드 레이아웃 재설계
   - 판정/신뢰도/사유/근거경로/액션 버튼의 정보 계층을 명확화.
   - `VIOLATE`, `UNKNOWN`, `OK` 상태별 시각 언어를 통일.

5. 배경 점검 상태 패널
   - 변경 구간 병합 결과(실행 단위/대기 건수/최근 완료 시간) 상시 노출.
   - 현재는 텍스트 상태만 제공하므로, 패널형 UI로 승격 필요.

6. 모바일 대응 레이아웃
   - 사이드바/편집기/결과패널 우선순위 재배치.
   - 터치 환경에서 카드 액션과 포커스 이동 UX 보강.

## 수용 기준(후속 과제 완료 정의)
- 각 위젯이 기존 API 계약(`/jobs`, `/jobs/{id}/events`, `/query/verdicts`)과 호환될 것.
- 키보드/모바일 접근성 기본 동작(포커스, 스크롤, 버튼 터치 영역) 보장.
- 기존 기능 회귀 없이 `pytest -q` 및 수동 스모크(저장-백그라운드 점검-상세근거) 통과.
