# UI 디자인 후속 요구사항 (2026-02-24)

## 범위와 원칙
- 본 문서는 신규 위젯/레이아웃 등 디자인 변경이 필요한 항목만 다룬다.
- 본 구현 사이클의 비디자인 코드 보완과 분리된 후속 과제로 관리한다.
- 기능 계약(API/DTO/DB)은 이미 마련된 경로를 재사용하고, 시각/상호작용만 확장한다.

## 후속 과제 목록
1. 세계관 시간축 타임라인 위젯
- 목적: `TIMELINE` 탭을 문서 트리 뷰가 아닌 세계관 시점 중심 뷰로 전환.
- 요구사항:
  - 시간축(연/월/일 또는 에피소드 상대 순서) 기반 시각화.
  - 에피소드/사건/앵커를 축 상에서 탐색 및 점프 가능.
  - `timeline_idx`, `time_key`와 양방향 연동.

2. 드래그 기반 태깅 위젯
- 목적: 편집 중 span 선택 후 복선/사건/인물 관련 태그를 즉시 부여.
- 요구사항:
  - 에디터 텍스트 선택 시 플로팅 액션 노출.
  - 프리셋 태그 + 사용자 정의 태그 선택.
  - 저장 시 span(start/end) + tag_path + 문맥 메타데이터 포함.

3. 인라인 플로팅 액션
- 목적: 정합성 결과에서 “예외 처리/무시/근거 이동”을 본문 맥락에서 빠르게 수행.
- 요구사항:
  - 하이라이트 클릭 시 카드형 액션 팝오버.
  - `whitelist_annotation(intent_type/reason/meta)` 입력 UI 연동.
  - 근거 문서/스팬으로 점프하는 deep-link 동작.

4. Jobs/Consistency 패널 시각 고도화
- 목적: 현재 텍스트 중심 상태 표현을 시각적으로 명확화.
- 요구사항:
  - 상태별 진행률/아이콘/경고 레벨 일관 표현.
  - 백그라운드 점검 큐(대기/처리/완료) 구분 표시.
  - 실패 시 재시도 액션과 오류 원인 요약 제공.

## UX 수용 기준 (후속)
1. 사용자 행동 수 최소화
- “문장 선택 → 태그 부여”를 2클릭 이내로 완료.

2. 컨텍스트 보존
- 에디터 포커스를 잃지 않고 예외 처리/근거 이동 가능.

3. 오판 대비
- `VIOLATE`, `UNKNOWN`, `whitelist`, `ignore`를 시각적으로 명확히 구분.

4. 반응성
- 데스크톱/모바일에서 레이아웃 깨짐 없이 동작.
- 위젯 오픈/닫힘 애니메이션은 과하지 않고 의미 중심으로 제공.

## 의존 관계
- 본 후속은 아래 비디자인 선행 반영을 전제로 한다.
  - `GET /jobs`, `GET /jobs/{job_id}`, `GET /jobs/{job_id}/artifact`
  - `whitelist_annotation` 저장/조회 계약
  - consistency 필터/고급 제어 파라미터 계약
