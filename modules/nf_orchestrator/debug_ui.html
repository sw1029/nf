작업

사용 방법

목적: 백그라운드 잡을 큐에 넣고 진행/결과(이벤트)를 확인합니다.

입력: 유형 + 프로젝트 ID + 입력/파라미터(JSON 객체)

출력: /jobs 응답(JSON) + /jobs/{job_id}/events SSE 이벤트

  • 유형/프로젝트 ID를 채우고 입력/파라미터(JSON)을 넣어 제출합니다.
  • 조회는 상태 폴링, 취소는 취소 요청을 전송합니다.
  • 실시간 이벤트는 SSE 뷰어에서 확인합니다(level, progress, payload).

입력(JSON) 예시 (유형별):

{
  "doc_id": "doc_id_here",
  "snapshot_id": "snapshot_id_here"
}
{
  "scope": "global",
  "snapshot_id": "snapshot_id_optional"
}
{
  "scope": "global",
  "shard_policy": {}
}
{
  "input_doc_id": "doc_id_here",
  "input_snapshot_id": "snapshot_id_here",
  "range": { "start": 0, "end": 1000 },
  "schema_ver": ""
}
{
  "query": "검색어",
  "filters": { "tag_path": "setting/character/name" },
  "k": 10
}
{
  "mode": "LOCAL_RULE",
  "range": { "doc_id": "doc_id_here", "snapshot_id": "snapshot_id_here", "start": 0, "end": 300 },
  "claim_text": "주장 텍스트",
  "include_citations": false
}
{
  "range": { "doc_id": "doc_id_here", "snapshot_id": "snapshot_id_optional" },
  "format": "txt",
  "include_meta": false
}

파라미터(JSON) 예시:

{
  "grouping": {
    "entity_mentions": true,
    "time_anchors": true,
    "timeline_doc_id": "doc_id_optional"
  }
}

출력(payload) 기대값(대표):

  • INGEST: schema_ver, proposed_fact_count, approved_fact_count
  • INDEX_FTS: chunks_indexed, entity_mentions_created, time_anchors_created, timeline_events_created
  • INDEX_VEC: shards_built
  • CONSISTENCY: vid_count, violate_count, unknown_count
  • RETRIEVE_VEC: 이벤트 다회 발생(results, page)
  • SUGGEST: suggestion_id, text, citations
  • PROOFREAD: lint_items[]
  • EXPORT: artifact_path, size_bytes
제출된 작업이 없습니다.

SSE 뷰어

사용 방법

목적: 잡 이벤트 스트림(text/event-stream)을 구독해 진행률/로그/결과를 실시간 확인합니다.

입력: job_id, (선택) 마지막 이벤트 ID

출력: SSE 이벤트(id=seq, data=JobEvent JSON)

  • 작업 ID를 입력하고 연결하면 이벤트 스트림을 구독합니다.
  • 끊김 후 마지막 이벤트 ID를 지정하고 재연결하면 이어받기가 가능합니다.
  • 페이로드 복사로 마지막 이벤트 JSON을 클립보드에 복사합니다.

JobEvent(요약):

{
  "level": "INFO|WARN|ERROR|PROGRESS",
  "message": "텍스트",
  "progress": 0.0,
  "payload": {}
}

검색(FTS 전용)

사용 방법

목적: FTS 동기 검색을 수행하고(필요 시) 벡터 검색을 비동기로 확장합니다.

입력: 쿼리, k, 필터(JSON 객체)

출력: 검색 결과(results[]) 및 벡터 확장 시 SSE 이벤트 스트림

  • 검색 실행은 FTS 동기 검색 결과를 표시합니다.
  • 벡터 확장은 비동기 벡터 검색을 시작하며, 벡터 스트림 중지로 중단합니다.
  • 검색 결과 기반으로 타임 앵커/엔티티 멘션 생성을 트리거할 수 있습니다(옵션: timeline_doc_id).

필터(JSON) 지원 키(대표):

  • tag_path, section, episode (FTS 조건)
  • entity_id, time_key, timeline_idx (메타 기반 후필터)
  • doc_id (벡터 확장 시 shard 선택에만 사용)

필터(JSON) 예시:

{
  "tag_path": "setting/character/name",
  "entity_id": "entity_id_optional",
  "timeline_idx": 1
}

제안

참고: API/LOCAL_GEN 모드는 서버/프로젝트 설정에서 비활성화되어 있으면 정책 위반으로 거부됩니다.
